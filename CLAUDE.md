# 小説プロジェクト「虚ろの子」

## プロジェクト概要

「空白の命刻」を持つ主人公・刻夜が、すべてを奪われた絶望から覚醒し、歪んだ世界の理を書き換えていく物語。陰陽五行をベースにした命刻システムが特徴。

## 作業別の参照ファイル

### 執筆時
1. **必須**: plot/guides/第X章.md（該当章のガイド）
2. 執筆ガイドは自己完結型。基本的にそのファイルだけで執筆可能

### 執筆ガイド作成時
**必ずタスク化すること**。以下のステップを順に実行：

1. [ ] settings/配下設定.md「本編タイムライン」を確認
2. [ ] 状態スナップショットを記入（テンプレート参照）
3. [ ] 構成提案（話数・タイトル・内容）→ **ユーザー承認待ち**
4. [ ] 承認後、ガイド本文作成（plot/guides/_テンプレート.md に沿う）
5. [ ] **登場キャラの性別・外見を characters/ から転記**（推測で書かない）
6. [ ] 整合性チェック（禁止事項・伏線・タイムラインとの矛盾）

※構成提案の承認なしにガイド作成に入らないこと
※タスク化を怠ると矛盾が発生しやすい
※**キャラ設定は必ず設定ファイルから転記。名前や属性から推測しない**

#### 執筆ガイド作成の詳細手順

**ステップ1: 情報収集**
```
必ず読むファイル:
- settings/配下設定.md → タイムライン、状態確認
- plot/parts/第X部_プロット.md → この章の位置づけ
- plot/guides/_テンプレート.md → フォーマット確認
- 前章の執筆ガイド/サマリー → 引き継ぎ事項
```

**ステップ2: キャラクター設定の転記**
```
登場キャラごとに以下を必ずチェック:
1. characters/十大主星.md 冒頭の【早見表】→ 性別・外見・口調
2. characters/十大主星.md 各キャラセクション → 詳細設定（必要に応じて）
3. characters/主人公.md → 刻夜の現在の状態
```

**ステップ3: 整合性チェック項目**
```
- [ ] 禁止情報が含まれていないか（rules/整合性ルール.md セクション8）
- [ ] キャラの性別・外見が設定と一致しているか
- [ ] キャラの口調サンプルと一致しているか
- [ ] タイムラインと矛盾がないか
- [ ] 前章との連続性があるか
```

### 設定追加・修正時
1. rules/整合性ルール.md のセクション12を参照
2. 必ずSSOTファイルに追加/修正
3. 派生ファイルがあれば同時に更新

### 矛盾チェック時
1. **rules/整合性チェックリスト.md を使用**（実行手順付き）
2. rules/整合性ルール.md の全セクションを確認
3. 特に注意：玄冥≠刻夜（別人物）

### 執筆後の整合性チェック手順
```
1. rules/整合性チェックリスト.md を開く
2. 「クイックチェック」を最低限実施
3. 時間があれば全項目チェック
4. 問題があれば修正して再チェック
```

## 重要な設定ポイント

### 玄冥と刻夜の違い
| | 玄冥（千年前） | 刻夜（現在） |
|---|---|---|
| 封印 | なし（最初から空白露見） | あり（17年間隠されていた） |
| 選択 | 「統べる」 | 「繋ぐ」を選べる |

### 情報開示タイミング
- 玄冥の名前 → 第五部まで隠す
- 刻夜が玄冥の血筋 → 第三部〜第四部
- 星禍の真意 → 第四部

詳細は rules/整合性ルール.md セクション8を参照。

## ディレクトリ構成

```
小説/
├── CLAUDE.md          ← このファイル
├── 本編/              ← 執筆済みの本文
│   ├── プロローグ/
│   └── 第一部/
├── characters/        ← キャラクター設定
│   └── 十大主星.md    ← 冒頭に【早見表】あり
├── settings/          ← 世界設定（配下設定.md含む）
├── plot/              ← プロット関連
│   ├── overview/      ← 全体プロット
│   ├── parts/         ← 各部のプロット
│   ├── guides/        ← 執筆ガイド
│   ├── summaries/     ← 執筆済み話のサマリー
│   └── memos/         ← 作業メモ
├── timeline/          ← 年表
├── rules/             ← ルール集
│   ├── 整合性ルール.md
│   ├── 整合性チェックリスト.md ← 執筆後チェック用
│   └── プロンプト集.md ← AI依頼用テンプレート
└── 資料/              ← 参考資料
```
