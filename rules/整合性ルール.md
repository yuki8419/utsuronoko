## 2. 封印設定の整理

### 玄冥（千年前）
- **封印なし**
- 生まれた時から空白の命刻が露見
- 最初から迫害を受けた
- プロローグの描写はこの設定に基づく

### 刻夜（現在）
- **星禍による封印あり**
- 生まれた時は空白だったが、星禍が封印
- 17年間「普通の命刻」に見えていた
- 村人は空白だと知らなかった
- 封印解除で初めて空白が露見→迫害開始

## 3. 空白の命刻の描写

### 正式な描写
```
左手の甲に「空白の命刻」——命刻が刻まれるべき場所に、
薄く光る円形の痕跡だけが残る（文字や紋様は一切なし）
```

### 禁止表現
- ❌「薄い灰色の印」
- ❌「何も見えない」（光る痕跡はある）
- ❌「真っ白」

## 6. 命刻表記ルール

### 正式表記
```
十干 + 十二支（例：庚丑、癸卯、丙午）
```

### 読み方を併記する場合
```
庚丑（かのえうし）——金の牛
癸卯（みずのとう）——水の兎
```

### 禁止表記
- ❌「庚丁E」のようなアルファベット混在
- ❌ 十干と十二支の順序逆転

## 12. 設定追加・修正のルール

### 設定追加の手順

1. **SSOTファイルに追加する**
   - 必ず「正となるファイル」（セクション4参照）に追加
   - 派生ファイル（執筆ガイド等）に先に書かない

2. **既存設定との整合性を確認**
   - 追加する設定が既存設定と矛盾しないか確認
   - 特に時系列（年表.md）との整合性をチェック

3. **関連ファイルへの反映**
   - 関連する執筆ガイドがあれば更新を検討
   - ただしSSOTが「正」であることを忘れない

### 設定修正の手順

1. **修正理由を明確にする**
   - なぜ修正が必要か（矛盾発見、ストーリー上の必要性など）

2. **SSOTファイルを修正**
   - 派生ファイルより先にSSOTを修正

3. **派生ファイルの同時修正**
   - 執筆ガイドなどに同じ記述がある場合、同時に修正
   - 修正漏れがないようにGrepで検索

4. **本編への影響を確認**
   - 既に執筆済みの本編に矛盾が生じないか確認
   - 生じる場合は本編修正 or 設定修正を再検討

### 修正時の注意事項

| 状況 | 対応 |
|------|------|
| 本編に既に書かれている設定を変更 | 本編も修正するか、設定変更を諦める |
| 複数ファイルに同じ設定がある | 必ずSSOTから修正し、派生を追従 |
| 曖昧だった設定を明確化 | 既存描写と矛盾しない形で追加 |
| 矛盾する設定を発見 | SSOTを「正」として他を修正 |

### 設定追加・修正のチェックリスト

- [ ] SSOTファイルに追加/修正したか
- [ ] 既存設定との矛盾がないか確認したか

- [ ] 派生ファイルに同じ記述があれば更新したか
- [ ] 執筆済みの本編に影響がないか確認したか
